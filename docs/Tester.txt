## 1. Общая концепция

### 1.1 Назначение
Создать инструмент для автоматического обнаружения и запуска тестов в проекте Markoff. Тестер реализуется как самостоятельная утилита, расположенная в `utils/Tester/`, и используется для тестирования как клиентской, так и серверной частей проекта.

### 1.2 Ключевые принципы

| Принцип | Описание |
|---------|----------|
| **Утилитарность** | Тестер — отдельный инструмент в экосистеме утилит проекта |
| **Предсказуемость** | Каждый тест изолирован, не влияет на другие |
| **Прозрачность** | Полная видимость процесса и результатов тестирования |
| **Масштабируемость** | Добавление тестов не требует изменения кода тестера |

```python
"""
Структуры данных для представления тестов.
Все датаклассы неизменяемые (frozen=True) для безопасности.
"""

@dataclass(frozen=True)
class TestFunction:
    """Отдельная тестовая функция"""
    name: str                    # Имя функции
    description: str             # Первая строка docstring
    module_path: str              # Путь к модулю (для импорта)
    file_path: Path               # Полный путь к файлу
    line_number: int              # Номер строки

@dataclass(frozen=True)
class TestNode:
    """Узел в дереве тестов"""
    name: str
    path: Path
    is_dir: bool
    parent: Optional['TestNode']
    children: tuple['TestNode', ...]  # Неизменяемый список
    test_functions: tuple[TestFunction, ...]
    
    def get_full_path(self) -> str:
        """Формирует путь вида client.core.bus"""
        parts = []
        current = self
        while current and current.name != "tests":
            parts.append(current.name)
            current = current.parent
        return ".".join(reversed(parts))
```

### 3.7 База для тестов (`common/test_common.py`)

```python
"""
Единая база для всех тестов проекта.
Все тесты должны импортировать: from utils.Tester.common.test_common import *
"""

# Реэкспорт основных утилит
from utils.logger import get_logger, Logger
from utils.Tester.utils.isolation import reset_environment, set_deterministic_mode

# Базовые классы для тестов
class TestHandler:
    """Базовый обработчик событий с подсчетом"""
    
    def __init__(self, name: str):
        self.name = name
        self.events = []
        self.logger = get_logger(f"TestHandler.{name}")
    
    def handle(self, event):
        self.events.append(event)
    
    def count(self, event_type: str = None) -> int:
        if event_type:
            return sum(1 for e in self.events if e.get('type') == event_type)
        return len(self.events)
    
    def clear(self):
        self.events.clear()

# Утилиты для asserts
def assert_event_count(handler: TestHandler, expected: int, event_type: str = None):
    """Проверяет количество полученных событий"""
    actual = handler.count(event_type)
    assert actual == expected, \
        f"Ожидалось {expected} событий {event_type or 'любых'}, получено {actual}"
```

### 4.2 Структура тестового файла

```python
"""
Модуль тестирования [название компонента].

Содержит тесты:
- test_имя1 - описание
- test_имя2 - описание
"""

from utils.Tester.common.test_common import *
from client.src.core.event_bus import EventBus

# Локальный логгер
logger = get_logger(__name__)

# Константы для тестов
TEST_EVENT = "test.event"

def test_basic_subscription():
    """
    Проверка базовой подписки и получения события.
    
    Последовательность:
    1. Создать обработчик
    2. Подписаться на событие
    3. Опубликовать событие
    4. Проверить получение
    """
    logger.info("Запуск теста базовой подписки")
    
    # Arrange
    bus = EventBus()
    handler = TestHandler("test")
    bus.subscribe(TEST_EVENT, handler.handle)
    
    # Act
    bus.emit(TEST_EVENT, {"data": 42})
    
    # Assert
    assert handler.count() == 1, "Событие не получено"
```

## 7. Обработка ошибок

### 7.1 Классификация и реакция

| Тип ошибки | Пример | Реакция |
|------------|--------|---------|
| **AssertionError** | `assert x == y` | Провал теста, сбор контекста |
| **TimeoutError** | Тест выполняется >10с | Принудительное прерывание |
| **ImportError** | Модуль не найден | Пропуск файла, предупреждение |
| **Exception** | Непредвиденная ошибка | Полный traceback |

### 7.2 Graceful Shutdown

```python
def setup_signal_handlers():
    """Обработка Ctrl+C и других сигналов"""
    
    def handler(signum, frame):
        print("\n\n⚠️  Получен сигнал прерывания")
        print("Завершаем текущий тест...")
        # Сохраняем частичные результаты
        # Чистим ресурсы
        sys.exit(0)
    
    signal.signal(signal.SIGINT, handler)
```

## 8. Критерии готовности

### 8.1 Функциональные требования

- [ ] Обнаружение всех тестов в `tests/` (рекурсивно)
- [ ] Навигация по папкам (цифры 1..N)

- [ ] Таймауты работают на Windows
- [ ] Фиксированный seed для воспроизводимости
- [ ] Обработка Ctrl+C

### 8.3 Требования к тестам

- [ ] Каждый тест имеет docstring
- [ ] Нет зависимостей между тестами
